# Скрипт для настройки и запуска окружения в Docker

Этот репозиторий предоставляет набор инструментов для автоматической настройки python окружения с использованием Docker. Скрипты и конфигурации предназначены для создания изолированного рабочего пространства для разработки, тестирования и запуска приложений в Docker-контейнерах.

## Структура репозитория

- `Dockerfile`: конфигурация для создания Docker-образа, который устанавливает все необходимые зависимости для python проекта.
- `start.sh`: скрипт для запуска и настройки контейнера, который запускает проект внутри Docker окружения.

## Зачем нужен этот скрипт

Этот проект помогает настроить полностью изолированное окружение для python проекта, используя Docker. Скрипт "start.sh" автоматически настроит и запустит ваш проект в контейнере, что обеспечит его работоспособность на любых системах, без необходимости установки зависимостей вручную.

Вместо того чтобы вручную настраивать виртуальные окружения и устанавливать зависимости, вы можете просто использовать Docker, чтобы запустить готовое окружение с одной командой.

### Задачи, которые выполняет скрипт:
1. Создание Docker-образа: С помощью "Dockerfile" вы создаете образ с установленным Python и всеми зависимостями.
2. Автоматический запуск окружения: Скрипт "start.sh" помогает автоматически запустить проект внутри Docker-контейнера, обеспечивая изоляцию и корректную работу всех зависимостей.
3. Управление зависимостями: Docker автоматически загрузит все необходимые пакеты и инструменты, определенные в проекте.

## Технические требования

1. Docker: eбедитесь, что у вас установлен Docker, так как все операции происходят в контейнерах.

2. Система:
   - Linux, Windows, или macOS.
   - Для работы с "start.sh" потребуется Bash.

3. Права доступа:
   - Убедитесь, что у вас есть права для запуска Docker и использования скриптов.

## Установка и использование

1. Клонирование репозитория

Склонируйте этот репозиторий на вашу локальную машину:

```
git clone https://github.com/ваш_проект.git
cd ваш_проект
```

2. Построение Docker-образа

Соберите Docker-образ.

3. Запуск контейнера.

После того как образ будет построен, вы можете запустить контейнер.
Контейнер запустит скрипт start.sh, который создаст виртуальное окружение и запустит проект.
